# Поставки — учёт отгрузок

Личный инструмент учёта поставок (маршрут, груз, даты, план/факт по деньгам в разных валютах). Один файл — `index.html`, ничего дополнительно ставить не нужно.

## Как разместить на GitHub Pages

1. Создайте новый репозиторий.
2. Загрузите `index.html` в корень репозитория.
3. Зайдите в **Settings → Pages**, в разделе Source выберите ветку `main` и папку `/root`, сохраните.
4. Через минуту-две сайт будет доступен по адресу вида `https://<ваш-логин>.github.io/<название-репозитория>/`.

## Хранение данных

- **Локально**: данные сохраняются в `localStorage` браузера.
- **В облаке**: кнопки **«☁ Сохранить»** и **«☁ Загрузить»** — данные синхронизируются через `data/shipments.json` в этом же репозитории (GitHub API). При загрузке страницы данные автоматически загружаются из облака.
- Любой, у кого есть ссылка, видит одни и те же данные (общее облачное хранилище).
- Регулярно нажимайте **«Excel»** для бэкапа на случай, если нужно восстановить более старую версию.

## Пароль входа

Пароль (`9761`) хранится в виде SHA-256 хеша, сам пароль в коде не фигурирует. При входе введённый пароль хешируется и сравнивается с хешем — даже открыв исходный код, пароль не прочитать. Это защита от случайного взгляда, но не от целевого анализа (атакующий может просто удалить проверку пароля через DevTools, так как весь код на клиенте).

## Файлы

- `index.html` — вся страница целиком (разметка, стили, логика). Ничего больше грузить не нужно, библиотека для Excel (`SheetJS`) подключается по ссылке из интернета автоматически.
